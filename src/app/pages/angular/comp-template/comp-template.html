Component

A component is a fundamental building block that controls a patch of the screen called a view. It is defined by the @Component() decorator and includes: TypeScript Class: This class contains the application data (properties) and logic (methods), such as handling user input and fetching data from a server. Metadata: The @Component decorator provides configuration information to Angular, including: selector: A CSS selector that tells Angular where to insert the component instance in the parent HTML (e.g., "app-root"). templateUrl or template: A reference to an external HTML file or an inline HTML string that defines the component's UI. styleUrls or styles: An array of external CSS file paths or inline CSS strings that apply styles specifically to this component's view. imports: (For standalone components) An array of dependencies (other components, directives, or pipes) that the template uses

Angular Concepts

  • Data Binding: A mechanism to coordinate data between the component class and the DOM.

  • ([property]="..."): Binds a component property to an HTML element's property.
  • Event Binding ((event)="..."): Responds to user events (such as clicks or input changes) by calling a method in the component class.
  • Two-way Data Binding

  • Directives: Instructions that modify the DOM structure or behavior.
  • Structural Directives: Alter the layout by adding, removing, or replacing elements in the DOM (e.g., *ngIf, *ngFor).
  • Attribute Directives: Alter the appearance or behavior of existing elements (e.g., ngModel, ngStyle).
